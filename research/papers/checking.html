The Gap

Embeddings approximate.
Some queries need precision.

CLIP and its descendants are extraordinary at vibes. They are unreliable at constraints — counts, identities, proportions, spatial relations, and text on a sign.

A vector search asks one fuzzy question: does this image, as a whole, feel close to this sentence, as a whole? That works beautifully for "a dog on a beach." It quietly fails for "two giraffes while a third eats from a tree," or "a sign that reads Norfolk," because the dominant visual feature drowns out the secondary constraints. The result is confident but unaccountable: you get a ranked list with no statement of which part of your request it actually honoured.

query  "A desk with two computer monitors, two mice, a cup and a keyboard on it."

An embedding model scores that whole sentence against a whole image. If the picture has one monitor and three cups, it may still rank highly — the global match is "close enough." The paper's wager is that we can do better by refusing to collapse the query into a single point, and instead treating every noun and relation as a claim that must be checked.

Instead of accepting uncertainty as a given, the framework explicitly verifies each atomic truth in the query against the retrieved content — marking what is satisfied and what remains unmet.
The Idea

Borrowing model checking
from formal verification

In hardware/software verification, a specification is a logical formula and you machine-check whether a system satisfies it. The paper maps that whole apparatus onto vision.

A query q becomes a system specification. A parser P(q)=φ breaks it into subject–predicate–object triplets — a graph of independent claims. A synthesiser M:φ→Π turns each triplet into a visual routine πᵢ: a small executable program that returns True/False on any image. An image v satisfies φ only if every routine fires True.

The lineage matters: like ViperGPT and VisProg, an LLM writes Python that calls perception APIs. Unlike them, the goal is not to answer one question but to build a reusable set of logical triplets and rank images by how well they satisfy it — using open-source models (Phi-4 for parsing & synthesis, OWL-V2 for detection) for reproducibility.

Granularity & state explosion. One could sample many subgraphs from φ — up to the cartesian product of all constraints, which blows up combinatorially. The paper sidesteps this the same way classical model checking does: compositional reasoning. A global statement is taken as true when its individual triplets are true, so each routine is verified locally and the results are aggregated, never enumerated jointly.

From Boolean to Ranking

Partial truth, and re-ranking

Verification is binary, but retrieval needs an ordering. The fix: rank by the proportion of routines satisfied — and optionally fold that into an existing embedding ranker as a re-ranker.

For the top-K of a base ranker, each image at position i gets a retrieval weight K − i, scaled by the fraction of verified triplets. The verifiable signal lifts correct images that the embedding under-ranked, without throwing away the embedding's recall.

Why this is honest. An image that scored low on the embedding but verifies 6/6 constraints gets promoted; one that scored high but only verifies 2/6 gets pushed down. The ordering is now traceable to a count of satisfied claims rather than an inscrutable cosine similarity.

The point of the taxonomy. A failure is never just "wrong." It is a bad triplet (parser), a bad code generation (synthesiser), a bad code execution (detector under-performs), or even a wrong annotation (the dataset, not the model). In the failure case from the paper, the detector mistook skis for a baseball bat — and because that decision is explicit, the wrong promotion is fully explainable. CLIP made a similar mistake on a tennis racket, but only a human guessing could say so.

04 · LIMITS & ETHICS
Honest about doubt
Synthesis/detection noise can offset gains. Routines for people, named entities, or sensitive content may inherit LLM bias — left to future work.

Future work points at named entities and real-world event context. But the core move is durable: anchoring image search in logic rather than intuition, so that "this matches" can finally be accompanied by "and here is exactly why."
