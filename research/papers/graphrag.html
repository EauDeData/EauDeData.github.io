The Tension

Flexible LLMs hallucinate.
Grounded graphs break on noise.

Archives need accountability — a guarantee that a fact is present or absent. LLMs give flexibility but invent. Knowledge graphs give grounding but assume clean text. Historical handwriting gives neither.

A plain RAG retrieves plausible passages and lets a generative model phrase an answer — convenient, but prone to hallucination and blind to anything outside its retrieved window. A knowledge graph is the opposite: every entity and relation is explicit and queryable. The catch is construction: when the graph is built from handwritten documents, recognition errors aren't rare glitches — they are a consistent characteristic of the data, splitting one entity into many misspelled ghosts and producing endless false negatives.

exact match  {name: 'girona'}  misses  gerona · gran · hieronma  — the same place, recognised three wrong ways.

The paper's answer is a semi-symbolic agent: keep the LLM's generalization for turning questions into queries, but ground execution in a graph — and, at runtime, relax the brittle string literals (names, places) into the set of variants word spotting actually found on the page.

The interleaved collaboration between word spotting and code generation lets the agent build queries robust to misinterpretation and hallucination, while still using approximate search where noise would otherwise hinder precise retrieval.

Property graph, or RDF?

Two ways to model the same Esposalles marriage record. The property graph is compact; the RDF version explodes names, surnames, and occupations into their own nodes — more nodes, but far more expressive for complex, multi-hop queries. Toggle between them.

Interactive · The Expansion Operator

One literal, all its ghosts

When the LLM marks a literal for approximate matching, word spotting returns its variants and the query is rewritten as a case-insensitive regex over the disjunction: l ↦ regex(v₁ ∨ … ∨ vₖ). Type a name or place and watch it expand.

literal marked for expansion girona
word-spotting variants
expanded Cypher literal
Try the paper's cases: girona, sabadell, carpenter, colomer — or any word, and orthographic OCR variants are generated for it.

This is the noise filter at query time. Exact matching for girona returns 5 individuals; PHOC expansion to (girona|gerona|gran|hieronma) recovers all 9 — the difference between a confidently wrong count and the right one.
